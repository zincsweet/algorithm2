# P1303 A*B Problem

## 题目背景

高精度乘法模板题。

## 题目描述

给出两个非负整数，求它们的乘积。

## 输入格式

输入共两行，每行一个非负整数。

## 输出格式

输出一个非负整数表示乘积。

## 输入输出样例 #1

### 输入 #1

```
1 
2
```

### 输出 #1

```
2
```

## 说明/提示

每个非负整数不超过 $10^{2000}$。

```c++
#include <bits/stdc++.h>
using namespace std;
const int N = 1e6 + 10;
int a[N], b[N], ans[N];
int la, lb, lans;

void mul(int a[], int b[], int ans[]) {
    for (int i = 0; i < lb; ++i) {
        for (int j = 0; j < la; ++j) {
            ans[i + j] += b[i] * a[j];
        }
    }
    for (int k = 0; k < lans; ++k) {
        ans[k + 1] += ans[k] / 10;
        ans[k] %= 10;
    }
    while (ans[lans - 1] == 0) --lans;
}
int main() {
    string sa, sb; cin >> sa >> sb;
    la = sa.size(); lb = sb.size(); lans = la + lb;
    if ((la == 1 && sa[0] == '0') || (lb == 1 && sb[0] == '0')) {
        cout << 0;
        return 0;
    }
    for (int i = 0; i < la; ++i) a[la - i -1] = sa[i] - '0';
    for (int i = 0; i < lb; ++i) b[lb - i -1] = sb[i] - '0';
    mul(a, b, ans);
    for (int i = lans - 1; i >= 0; --i) {
        cout << ans[i];
    }
    return 0;
}
```

## 一、理解“无进位相乘 + 最后处理进位”的优化思想

传统竖式乘法每乘完一位就要处理进位，代码会变得复杂。而视频中介绍的**无进位相乘**方法极大地简化了逻辑：

1. 先不管进位，把所有位相乘的结果累加到对应位置（利用下标规律 `ans[i+j] += a[i] * b[j]`）。
2. 所有乘积累加完成后，**再统一从低位到高位处理进位**。

这样做的优点：

- 代码更简洁，只需两层循环累加，再一层循环处理进位。
- 避免了在每步乘法中频繁进行除法和取模运算，效率更高。

> **收获**：学会将复杂过程拆分成多个简单阶段，先完成核心计算，再处理附加规则（如进位），这是算法优化的重要思路。

------

## 二、掌握逆序存储下的下标对应规律

因为两个乘数都逆序存储（个位在数组下标0），所以：

- 第一个数的第 `i` 位（对应 `10^i` 权值）和第二个数的第 `j` 位（对应 `10^j` 权值）相乘，结果应累加到 `ans[i+j]` 位置（对应 `10^{i+j}` 权值）。

- 这个规律非常直观，代码实现就是简单的二重循环：

	cpp

	```
	for (int i = 0; i < lb; ++i)
	    for (int j = 0; j < la; ++j)
	        ans[i + j] += b[i] * a[j];
	```

	

> **收获**：逆序存储让“位权”与“数组下标”直接对应，极大简化了高精度运算的代码设计。这个规律会在以后的高精度乘方、卷积等场景中反复使用。

------

## 三、处理结果为 0 的特殊情况

如果两个数中有一个是 0，则结果直接为 0。代码中单独判断：

cpp

```
if ((la == 1 && sa[0] == '0') || (lb == 1 && sb[0] == '0')) {
    cout << 0;
    return 0;
}
```



这是非常必要的，因为后续的 `while (ans[lans - 1] == 0) --lans;` 虽然也能去掉前导零，但如果结果全是零，会一直减到 `lans` 变成 0，导致输出为空（错误）。所以提前处理保证结果至少有一个 `0` 输出。

> **收获**：边界条件永远是高精度题目的重点，特别是涉及零、前导零、进位时，必须仔细考虑。

------

## 四、前导零的去除（与减法类似）

乘法结果可能高位有多个零（比如 `123 * 0` 或 `100 * 1`），需要用循环去掉：

cpp

```
while (ans[lans - 1] == 0) --lans;
```



注意：这个循环假设结果至少有一位非零，所以上面必须处理全零的情况。与前导零处理类似，体现了代码复用性。

> **收获**：高精度运算中，前导零去除是标准操作，要熟练掌握 `while (ans[lans-1]==0 && lans>1) --lans;` 这种写法。

------

## 五、结果位数的预估

乘法结果的最大位数是两个乘数的位数之和，所以 `lans = la + lb;` 是安全的。即使实际结果位数更少，后续前导零去除会修正长度。

> **收获**：在做高精度运算时，预先分配足够大的数组（`N = 1e6+10`）并合理预估结果长度，可以避免越界问题。
