# P1480 A/B Problem（高精度除法Ⅰ）

## 题目描述

输入两个整数 $a,b$，输出它们的商。

## 输入格式

两行，第一行是被除数，第二行是除数。

## 输出格式

一行，商的整数部分。

## 输入输出样例 #1

### 输入 #1

```
10
2
```

### 输出 #1

```
5
```

## 说明/提示

$0\le a\le 10^{5000}$，$1\le b\le 10^9$。

```c++
#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int N = 1e6 + 10;
int a[N], b, ans[N];
int la, lans;

void div(int a[], int b, int ans[]) {
    LL rem = 0;
    for (int i = la - 1; i >= 0; --i) {
        rem = rem * 10 + a[i];
        ans[i] = rem / b;
        rem %= b;
    }
    while (ans[lans - 1] == 0 && lans > 1) --lans;
}

int main() {
    string sa; cin >> sa >> b;
    la = sa.size(); lans = la;
    for (int i = 0; i < la; ++i) a[la - i -1] = sa[i] - '0';
    div(a, b, ans);
    for (int i = lans - 1; i >= 0; --i) {
        cout << ans[i];
    }
    return 0;
}
```

“高精度除法（**==高精度除以低精度==**）”能带给你的核心收获。这是高精度系列的收官之作，它的计算方向和处理方式与前三种运算完全不同，值得认真体会。

------

## 一、理解“从高位到低位”的除法模拟过程

与加减乘（都是从低位（个位）开始）不同，**除法竖式是从最高位开始试除的**。
代码中循环从 `la - 1` 到 `0`，正好对应从高位到低位：

cpp

```
for (int i = la - 1; i >= 0; --i) { ... }
```



这一步理解到位，就不会把加减乘的惯性带到除法里。

> **收获**：模拟竖式时，必须严格遵循手算的顺序。除法“从高位开始”是它的核心特征，也是与加减乘最大的区别。

------

## 二、余数变量 `rem` 的巧妙运用

模拟除法时，每一步都需要将上一步的余数 `rem` 与当前位 `a[i]` 组合成新的被除数：

cpp

```
rem = rem * 10 + a[i];    // 余数左移一位，加上当前位
ans[i] = rem / b;         // 得到当前位的商
rem %= b;                 // 更新余数
```



这个“滚动余数”的技巧，实际上模拟了手算中“落位”的过程，非常精妙。

> **收获**：学会用单个变量维护中间状态（余数），避免存储所有中间被除数，极大地节省了空间和代码量。

------

## 三、数据类型选择的重要性

虽然被除数是高精度（数组存储），但除数 `b` 是低精度（`int` 范围，`10^9` 级别）。余数 `rem` 可能达到 `10^9 * 10 + 9 ≈ 10^10`，超出了 `int` 的范围，所以必须用 `long long`。

cpp

```
long long rem = 0;
```



这是很多初学者容易忽略的细节，一旦溢出就会得到错误答案。

> **收获**：在混合精度运算中，要时刻关注中间结果的范围，选择合适的变量类型，避免溢出。

------

## 四、前导零处理的变体（与加减乘一致）

除法结果可能高位有多个零（例如 `100 / 5 = 020`，实际应为 `20`），需要去除前导零：

cpp

```
while (ans[lans - 1] == 0 && lans > 1) --lans;
```



注意保留至少一位，保证结果为 `0` 时能正确输出一个 `0`。

> **收获**：前导零处理是高精度运算的标配操作，要能熟练写出这个 `while` 循环，并理解 `lans > 1` 的保护条件。

------

## 五、结果位数与被除数相同（无需额外预估）

除法（高精除以低精）的结果位数最多与被除数位数相同，所以可以直接令 `lans = la;`，并在 `div` 函数中直接向 `ans[i]` 赋值，下标与高位对应。这比乘法需要 `la + lb` 的预估更简单。

> **收获**：根据运算类型合理预判结果长度，避免数组越界或不必要的扩容。

------

## 六、与加减乘的对比，形成完整的“高精度四则运算”框架

| 运算              | 遍历方向                 | 核心变量       | 特殊处理         |
| :---------------- | :----------------------- | :------------- | :--------------- |
| 加法              | 低位→高位（0→len）       | 进位 `carry`   | 最高位进位       |
| 减法              | 低位→高位（0→len）       | 借位标记（-1） | 符号、前导零     |
| 乘法              | 低位→高位（二重循环）    | 累加数组       | 统一进位、前导零 |
| 除法（高精/低精） | **高位→低位**（len-1→0） | 余数 `rem`     | 前导零           |
