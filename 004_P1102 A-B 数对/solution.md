# P1102 A-B 数对

## 题目描述

给出一串正整数数列以及一个正整数 $C$，要求计算出所有满足 $A - B = C$ 的数对的个数（不同位置的数字一样的数对算不同的数对）。

## 输入格式

输入共两行。

第一行，两个正整数 $N,C$。

第二行，第 $i$ 个数为 $a_i$，数字之间用一个空格隔开，共 $N$ 个正整数，作为要求处理的那串数。

## 输出格式

一行，表示该串正整数中包含的满足 $A - B = C$ 的数对的个数。

## 输入输出样例 #1

### 输入 #1

```
4 1
1 1 2 3

```

### 输出 #1

```
3
```

## 说明/提示

对于 $75\%$ 的数据，$1 \leq N \leq 2000$。

对于 $100\%$ 的数据，$1 \leq N \leq 2 \times 10^5$，$0 \leq a_i <2^{30}$，$1 \leq C < 2^{30}$。

2017/4/29 新添数据两组

```c++
#include <bits/stdc++.h>
using namespace std;
int main() {
    std::ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(nullptr);
    int n, c;
    long long ans = 0;
    int arr[200010];
    cin >> n >> c;
    for (int i = 1; i <= n; ++i) {
        cin >> arr[i];
    }
    sort(arr + 1, arr + 1 + n);
    int b = 1, a_l = 1, a_r = 1;
    for (b; b <= n; ++b) {
        while (a_r <= n && arr[a_r] - arr[b] <= c) ++a_r;
        while (a_l <= n && arr[a_l] - arr[b] < c) ++a_l;
        if (arr[a_l] - arr[b] == c && arr[a_r - 1] - arr[b] == c && a_r - 1 >= 1) {
            ans += a_r - a_l;
        }
    }
    cout << ans;
    return 0;
}
```

利用了**排序 + 双指针（尺取法）**，巧妙地将“数对计数”转化为“区间长度统计”。

### 一、核心原理（降维转化）

将等式 A−B=C*A*−*B*=*C* 转化为 **A=B+C\*A\*=\*B\*+\*C\***。
排序后，数组具有单调性。因此，对于每一个固定的 `B = arr[b]`，所有满足条件的 `A` 必然在数组中**连续排列**，且都等于 `arr[b] + C`。

### 二、双指针的精确语义（最关键）

定义了两个右指针，分别维护这个“目标值区间”的左右边界，且均为**左闭右开**风格：

- **`a_l`（左边界）**：指向**第一个**满足 `arr[a_l] - arr[b] >= C` 的位置（即目标值 B+C*B*+*C* 首次出现的位置）。
- **`a_r`（右边界）**：指向**第一个**满足 `arr[a_r] - arr[b] > C` 的位置（即目标值 B+C*B*+*C* 结束后的第一个位置）。

**指针移动逻辑**：

- `while (a_r <= n && arr[a_r] - arr[b] <= c) ++a_r;`
	只要差值没超过 C，`a_r` 就一直右移，最终停在 **> C** 处。
- `while (a_l <= n && arr[a_l] - arr[b] < c) ++a_l;`
	只要差值还小于 C，`a_l` 就一直右移，最终停在 **>= C** 处。

### 三、单调性与不回退（效率保证）

由于数组递增，当 `b` 右移（即 B*B* 变大）时，目标值 B+C*B*+*C* 也随之变大。
因此，`a_l` 和 `a_r` 在上一轮的位置，对于新的 `b` 依然有效（或偏左），**无需重置回起点**，两个指针在整个过程中只向右单向移动，实现了 O(N)*O*(*N*) 的扫描。

### 四、计数方法与边界细节

如果目标值存在，区间 `[a_l, a_r)` 内的所有元素都等于 B+C*B*+*C*，数量即为 `a_r - a_l`。

**`if` 判断很精髓**：

```
if (arr[a_l] - arr[b] == c && arr[a_r - 1] - arr[b] == c && a_r - 1 >= 1)
```

- 它严格确保 `a_l` 没有越界且恰好命中目标值；
- 同时通过 `a_r - 1 >= 1` 排除了因空区间导致 `a_r` 回退到 0 的风险，保证了下标 `a_r-1` 在有效数据范围内，兼顾了逻辑正确性与数组访问安全。

### 五、复杂度

- **时间复杂度**：O(Nlog⁡N)*O*(*N*log*N*)（主要来自排序） + O(N)*O*(*N*)（双指针扫描）。
- **空间复杂度**：O(N)*O*(*N*)。
