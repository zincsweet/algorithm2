# P1601 高精度加法

## 题目背景

本题是高精度加法的模板题。

## 题目描述

给定两个非负整数 $a,b$，求它们的和。**不用考虑负数**。

## 输入格式

输入共两行，每行一个非负整数，分别为 $a,b$。

## 输出格式

输出一行一个非负整数，表示 $a+b$ 的值。

## 输入输出样例 #1

### 输入 #1

```
1
1
```

### 输出 #1

```
2
```

## 输入输出样例 #2

### 输入 #2

```
1001
9099
```

### 输出 #2

```
10100
```

## 说明/提示

对于 $20\%$ 的测试数据，$a,b \le 10^9$；  
对于 $40\%$ 的测试数据，$a,b \le 10^{18}$；  
对于 $100\%$ 的测试数据，$0\le a,b \le 10^{500}$。

```c++
#include <bits/stdc++.h>
using namespace std;
const int N = 1e6 + 10;
int a[N], b[N], sum[N];
int la, lb, lsum;

void add(int a[], int b[], int sum[]) {
    for (int i = 0; i < lsum; ++i) {
        sum[i] += a[i] + b[i];
        sum[i + 1] = sum[i] / 10;
        sum[i] %= 10;
    }
    if (sum[lsum]) ++lsum;
}

int main() {
    string sa, sb;
    cin >> sa >> sb;
    // 1. 逆序字符串转换
    la = sa.size();
    lb = sb.size();
    lsum = max(la, lb);
    for (int i = 0; i < la; ++i) {
        a[la - i - 1] = sa[i] - '0';
    }
    for (int i = 0; i < lb; ++i) {
        b[lb - i - 1] = sb[i] - '0';
    }
    // 2. 竖式相加
    add(a, b, sum);
    // 3. 输出结果
    for (int i = lsum - 1; i >= 0; --i) {
        cout << sum[i];
    }
    return 0;
}
```

## 一、理解“大数存储”的底层逻辑

- **字符串读入**：当数据远超 `int` / `long long` 范围时，把它当成**字符串**读入，因为字符串可以任意长（内存允许即可）。
- **数组存储每一位**：字符串里是字符，用 `s[i] - '0'` 转成数字，再存入 `int` 数组。
- **逆序存储**：数组下标 0 存个位，1 存十位……这样从前往后遍历时，正好对应小学竖式从低位到高位的计算顺序，**进位也很容易处理**。

> 这一思维以后会反复用在高精度减法、乘法、除法中。

------

## 二、模拟“竖式计算”的过程

加法竖式本质上就是三步循环：

1. **对应位相加 + 进位**
2. **处理进位**（当前位 / 10 加到下一位）
3. **保留当前位**（当前位 % 10）

代码里这三行就对应了这个过程：

cpp

```
sum[i] += a[i] + b[i];      // 对应位相加，并加上之前的进位
sum[i+1] = sum[i] / 10;     // 进位给下一位
sum[i] %= 10;               // 当前位只保留个位
```



这个“模拟”思想是算法入门中非常重要的：**用代码去模拟我们手算时的每一步**。

------

## 三、处理边界 / 特殊情况

- **最高位进位**：比如 `99 + 1 = 100`，原本两个数最长是 2 位，但结果变成了 3 位。
	代码中通过 `if (sum[lsum]) ++lsum;` 处理，很简洁，但很容易遗漏，是很多同学的失分点。
- **数组下标与字符串下标的转换关系**：
	`a[la - i - 1] = sa[i] - '0';` 这个公式值得记住，它把正序字符串映射到逆序数组。

------

## 四、代码模块化与全局变量的运用

- 把加法过程单独封装成 `add` 函数，让主流程清晰。
- `la, lb, lsum` 作为全局变量，方便函数里直接修改 `lsum`（其实更规范的做法是传引用，但在竞赛中全局变量很常见）。
- 数组开成全局变量（`const int N = 1e6 + 10;`），能开得比较大，避免栈溢出。

------

## 五、养成“三步走”的解题框架

高精度类问题通常都可以按这个模板来：

1. **读入字符串**，求出长度
2. **逆序存入数组**
3. **模拟计算**（加法/减法/乘法/除法）
4. **逆序输出结果**

这种结构化思考方式，对你以后写复杂模拟题很有帮助。

------

## 六、为后续高精度算法打好基础

这道题是高精度的“第一个台阶”。理解了加法，减法就是处理借位，乘法就是循环乘加，除法就是模拟试商。它们都是在这个基础上扩展的，所以这次学扎实了，后面会轻松很多。

------

## 七、代码细节带来的收获

- 注意 `sum[i] += a[i] + b[i];` 一定要用 `+=`，因为 `sum[i]` 里可能已经有进位。
- 循环边界 `i < lsum`，`lsum` 取两个数长度的最大值，确保覆盖所有位数。
- 输出时从 `lsum - 1` 到 0，去除前导 0（这里没有前导 0，但以后可能有）。
