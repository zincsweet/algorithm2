# P2005 A/B Problem II

## 题目背景

为了让大家紧张的心情放松一下，这一题是一道非常简单的题目。

## 题目描述

给出正整数 $N$ 和 $M$，请你计算 $\lfloor\frac{N}{M}\rfloor$（$\frac{N}{M}$ 的下取整）。

## 输入格式

两行，两个正整数，$N$ 和 $M$。

## 输出格式

一行，一个整数，表示 $\lfloor\frac{N}{M}\rfloor$。

## 输入输出样例 #1

### 输入 #1

```
1000 
333

```

### 输出 #1

```
3

```

## 说明/提示

- 对于 $60 \%$ 的数据：$ N,M \le 750!$，答案 $ \le 7!$。
- 对于 $100 \%$ 的数据：$N,M \le 6250!$，答案 $ \le 13!$。

请注意：上面的两句话并不是感叹句，而是陈述句。标点符号使用没有错误。

```c++
#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int N = 1e6 + 10;
int a[N], b[N], tmp[N];
int la, lb, ltmp;

int cmp(int x[], int lx, int y[], int ly) {
    if (lx != ly) return lx > ly ? 1 : -1;
    for (int i = lx - 1; i >= 0; --i) {
        if (x[i] != y[i]) return x[i] > y[i] ? 1 : -1;
    }
    return 0;
}

bool lessOReq(int a[], int la, int b[], int lb, LL mid) {
    if (mid == 0) return true;
    ltmp = lb;
    LL carry = 0;
    for (int i = 0; i < lb ; ++i) {
        LL cur = 1LL * b[i] * mid + carry;
        carry = cur / 10;
        tmp[i] = cur % 10;
    }
    while (carry) {
        tmp[ltmp++] = carry % 10;
        carry /= 10;
    }
    return cmp(a, la, tmp, ltmp) >= 0;
}

int main() {
    string sa, sb; cin >> sa >> sb;
    la = sa.size(); lb = sb.size();
    for (int i = 0; i < la; ++i) a[i] = sa[la - i -1] - '0';
    for (int i = 0; i < lb; ++i) b[i] = sb[lb - i -1] - '0';
    
    LL left = 0, right = 1;
    for (int i = 1; i <= 13; ++i) {
        right *= i;
    }

    while (left < right) {
        LL mid = left + (right - left + 1) / 2;
        if (lessOReq(a, la, b, lb, mid)) {
            left = mid;
        }
        else {
            right = mid - 1;
        }
    }
    
    cout << left;
    return 0;
}
```

### 收获一：打破“见题就用模板”的惯性思维

这是你最大的成长！

- **如果机械套用模板**：你会用刚学会的“高精度÷高精度”竖式去算，代码复杂、容易出错、还可能在位数极大时超时。
- **你现在的做法**：观察到“商 ≤ 13!”这个关键条件，转而用**二分答案 + 高精度 × long long**，代码更短、更稳、更快。

> **核心启示**：算法竞赛里，**读懂题目限制比背模板更重要**。限制条件本身就是出题人留给你的“解题捷径”。

------

### 收获二：二分答案的通用模型（学会“猜答案”）

这道题帮你建立了一个极其实用的解题框架：

| 步骤               | 具体操作                               |
| :----------------- | :------------------------------------- |
| **① 确定答案范围** | 题目保证答案 ≤ 13!，所以 `[0, 13!]`    |
| **② 设计判断函数** | `check(mid)` 判断 `M × mid ≤ N`        |
| **③ 二分查找**     | 找最大的满足条件的 `mid`，就是下取整商 |

这个框架可以迁移到无数题目中（比如“求最大可行值”“最小可行值”）。你以后遇到“求满足某条件的最大/最小整数”时，会立刻想到二分。

------

### 收获三：高精度乘法的灵活运用（降维打击）

你原来写的 `mul_short`（高精度 × 个位数）是死板的。这道题迫使你把它**升级成“高精度 × long long”**：

cpp

```
for (int i = 0; i < lb; ++i) {
    LL cur = 1LL * b[i] * mid + carry;
    tmp[i] = cur % 10;
    carry = cur / 10;
}
```



你学会了：

- 用 `long long` 暂存中间结果，防止 `int` 溢出（这是你之前卡住的关键）。
- 进位可能很大，但 `while(carry)` 能自适应地扩展长度。
- 这个函数现在可以复用到任何“高精度 × 普通整数”的场景（比如高精度乘以 `1e9`、乘以 `10^18` 以内的数）。

------

### 收获四：边界与容量的深刻理解

你在这道题里踩过的坑，比看十篇博客都有用：

| 坑点                               | 你学会的教训                                   |
| :--------------------------------- | :--------------------------------------------- |
| `b[i] * mid` 赋值给 `int` 会截断   | 用 `long long` 存中间结果，取模后再转 `int`    |
| 进位循环写成 `carry / 10` 变死循环 | 进位拆解必须用 `carry % 10` 然后 `carry /= 10` |
| 右边界不能随便设                   | 根据题目保证设定，防止乘法溢出                 |
| 二分中间值 `(L+R)/2` 可能溢出      | 用 `L + (R-L+1)/2` 更安全                      |

这些经验是**无价的**，将来写其他代码时你会本能地避开这些陷阱。

------

### 收获五：理解“高精度÷高精度”与“二分答案”的适用场景

现在你可以清晰地区分两种策略：

| 场景                                     | 推荐做法                    | 原因                                 |
| :--------------------------------------- | :-------------------------- | :----------------------------------- |
| **被除数和除数都极大，商也极大**         | 高精度÷高精度（竖式模拟）   | 二分右边界无法确定，且乘法可能溢出   |
| **被除数和除数极大，但商很小（有上界）** | 二分答案 + 高精度乘法       | 二分次数少（log 级别），乘法简单高效 |
| **被除数极大，除数是个普通整数**         | 高精度÷低精度（模拟长除法） | 一行代码的事，效率最高               |

你以后看到一道题，会先问自己三个问题：

1. 答案有上界吗？
2. 这个上界大到无法乘法吗？
3. 是直接除还是二分？

这就是**算法设计意识**的萌芽。
