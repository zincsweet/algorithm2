# P5731 【深基5.习6】蛇形方阵

## 题目描述

给出一个不大于 $9$ 的正整数 $n$，输出 $n\times n$ 
的蛇形方阵。

从左上角填上 $1$ 开始，顺时针方向依次填入数字，如同样例所示。注意每个数字有都会占用 $3$ 个字符，前面使用空格补齐。

## 输入格式

输入一个正整数 $n$，含义如题所述。

## 输出格式

输出符合题目要求的蛇形矩阵。

## 输入输出样例 #1

### 输入 #1

```
4
```

### 输出 #1

```
  1  2  3  4
 12 13 14  5
 11 16 15  6
 10  9  8  7
```

## 说明/提示

数据保证，$1 \leq n \leq 9$。

```c++
#include <bits/stdc++.h>
using namespace std;
int main() {
            // 右,下,左,上
    int dx[] = {0, 1, 0, -1};
    int dy[] = {1, 0, -1, 0};
    int arr[11][11] = {};
    int n, num = 1;
    int x = 1, y = 1;    // 初始化x,y第一个位置
    int dir = 0;    // 起始方向
    cin >> n;
    while (num <= n * n) {
        arr[x][y] = num;
        int next_x = x + dx[dir], next_y = y + dy[dir];
        // 通过下一次的位置判断需不需要转向
        if (next_x < 1 || next_x > n || next_y < 1 || next_y > n || arr[next_x][next_y] != 0) {
            dir = (dir + 1) % 4;
            next_x = x + dx[dir];
            next_y = y + dy[dir];
        }
        // 更新位置
        x = next_x;
        y = next_y;
        ++num;
    }
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= n; ++j) {
            printf("%3d", arr[i][j]);
        }
        cout << endl;
    }
    return 0;
}
```

## 🎯 核心收获

### 1. **方向向量（偏移量数组）—— 矩阵类问题的王牌技巧**

这是这道题最重要的收获。通过两个数组 `dx[]` 和 `dy[]`，将“上下左右”四个方向的移动抽象成了**数组下标的变化**：

cpp

```
int dx[] = {0, 1, 0, -1};   // 右、下、左、上 的 X 偏移
int dy[] = {1, 0, -1, 0};   // 右、下、左、上 的 Y 偏移
```



**价值在于：**

- 原本需要写 4 个 `if` / 4 层循环的逻辑，现在只需 **一次数组取值 + 循环取模** 就能完成方向切换（`dir = (dir + 1) % 4`）。
- 代码量大幅减少，**出错率显著降低**。
- 这个技巧不局限于此题，在 **BFS/DFS 搜索、迷宫问题、岛屿问题** 中会反复用到（比如遍历相邻格子时，方向向量就是标准写法）。

------

### 2. **“预判下一步”的写法 —— 优雅的循环控制**

你的代码中采用了一种很干净的模式：

cpp

```
int next_x = x + dx[dir];
int next_y = y + dy[dir];
if (越界或已访问) {
    dir = (dir + 1) % 4;           // 换方向
    next_x = x + dx[dir];          // 重新计算下一步
}
x = next_x;  // 统一更新
y = next_y;
```



**相较于其他写法（比如先走再回退），这种方式的优点：**

- 当前位置 `(x, y)` 始终是“已经填过数”的合法位置。
- 只根据“下一步是否合法”来决定是否转向，**逻辑集中、容易理解**。
- 最后统一更新坐标，避免了在分支里重复写 `x = ... y = ...` 的冗余。

------

### 3. **模拟思维的强化**

这道题属于“纯模拟”类型 —— 题目已经给出了明确的填数规则，你只需要把规则翻译成代码即可。

**通过这道题你学会了：**

- 如何把一个**自然语言描述的过程**（“顺时针螺旋填数”）拆解成**明确的程序步骤**。
- 如何用一个 `while` 循环 + 状态变量（`dir`）来**持续执行这个重复过程**，直到填满整个矩阵。
- 这种“模拟过程直到结束”的思维，在后面的 **字符串处理、日期计算、游戏逻辑** 等题目中会反复用到。

------

### 4. **边界条件的处理能力**

在蛇形填数中，边界条件有 3 种情况：

- 行越界（`next_x < 1 || next_x > n`）
- 列越界（`next_y < 1 || next_y > n`）
- 目标位置已经填过数（`arr[next_x][next_y] != 0`）

你的代码把这三种情况统一放在一个 `if` 条件里处理，这种**统一处理边界**的方式能帮你避免遗漏情况，也便于后续维护。

------

### 5. **输出格式的规范化**

题目要求“每个数字占 3 个字符，右对齐”，使用 `printf("%3d", arr[i][j]);` 就是标准的 C 风格格式化输出。

**背后的道理：** 很多 OJ 题目对输出格式有严格要求（如场宽、补零、对齐），学会 `printf` 的格式控制符是必备技能。

------

### 6. **从零开始还是从一开始？—— 数组下标习惯的养成**

你的代码把 `x` 和 `y` 初始化为 `1`，数组也按 `[1..n]` 使用，第 0 行第 0 列弃用。

**这样做的好处：**

- 避免了在边界判断时出现 `-1` 或 `0` 的混乱，逻辑更直观。
- 这种“下标从 1 开始”的习惯，在**动态规划、前缀和、树状数组**等场景中也很常见（空出 0 位置作为哨兵）。

- 改动填充的值（数字 / 字符 / 自定义顺序）

这个框架就能直接复用。
