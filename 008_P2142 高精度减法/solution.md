# P2142 高精度减法

## 题目描述

给定两个正整数 $a,b$，求 $a-b$ 的值。

## 输入格式

输入共两行，每行一个正整数，分别为 $a,b$。

## 输出格式

输出一行一个整数，表示 $a-b$ 的值。

如果 $a-b<0$，**请输出负号**。

## 输入输出样例 #1

### 输入 #1

```
2
1
```

### 输出 #1

```
1
```

## 说明/提示

对于 $20\%$ 的数据，$a,b$ 在`long long`类型的存储范围内；  
对于 $100\%$ 的数据，$0<a,b\le 10^{10086}$。

```c++
#include <bits/stdc++.h>
using namespace std;
const int N = 1e6 + 10;
int a[N], b[N], ans[N];
int la, lb, lans;

bool cmp(string& sa, string& sb) {
    // if (sa.size() > sb.size()) {
    //     return false;
    // }
    // else if (sa.size() < sb.size()) {
    //     return true;
    // }
    // else {
    // }
    if (sa.size() != sb.size()) return sa.size() < sb.size();
    return sa < sb;
}

void sub(int a[], int b[], int ans[]) {
    for (int i = 0; i < lans; ++i) {
        ans[i] += a[i] - b[i];
        if (ans[i] < 0) {
            ans[i + 1] = -1;
            ans[i] += 10;
        }
    }
    while (ans[lans - 1] == 0 && lans > 1) --lans;
}

int main() {
    string sa, sb; cin >> sa >> sb;
    // 1. 判断减数大小
    if (cmp(sa, sb)) {
        swap(sa, sb);
        cout << '-';
    }
    // 2. 逆序存入
    la = sa.size(); lb = sb.size(); lans = max(la, lb);
    for (int i = 0; i < la; ++i) a[la - i - 1] = sa[i] - '0';
    for (int i = 0; i < lb; ++i) b[lb - i - 1] = sb[i] - '0';
    // 3. 竖式减法
    sub(a, b, ans);
    // 4. 输出
    for (int i = lans - 1; i >= 0; --i) cout << ans[i];
    
    return 0;
}
```

## 一、理解“借位”的模拟方式

减法竖式的核心是**借位**。代码中用短短几行就实现了：

cpp

```
ans[i] += a[i] - b[i];      // 当前位相减（并带上之前可能借来的 -1）
if (ans[i] < 0) {           // 不够减，需要向高位借位
    ans[i + 1] = -1;        // 高位标记借走 1
    ans[i] += 10;           // 当前位加 10
}
```



这个“借位标记”的设计非常巧妙：

- 不直接修改高位值，而是用 `-1` 表示“高位已被借走 1”，下一轮循环会加上这个 `-1`，相当于高位自动减 1。
- 这避免了复杂的前后依赖，逻辑清晰。

> **收获**：学会用“标记”代替“直接修改”，让模拟更简洁。

------

## 二、比较两个“大数”的正确方法

**不能直接用字符串比较**，因为字典序和数值大小可能不一致（如 `"99"` 和 `"100"`）。
正确做法：

1. **先比较长度**，长度大的数一定大。
2. 长度相同再按字典序比较。

代码中的 `cmp` 函数完美体现了这个原则：

cpp

```
if (sa.size() != sb.size()) return sa.size() < sb.size();
return sa < sb;
```



> **收获**：在处理字符串表示的数字时，永远先考虑长度，再考虑字典序。这是一个通用的技巧。

------

## 三、符号处理：大数减小数

减法必须保证“大数减小数”，如果输入是 `a < b`，则：

- 交换两个字符串
- 输出一个负号
- 然后计算 `b - a`

这样后续的模拟始终处理正数减法，逻辑统一，避免了符号混乱。

> **收获**：把复杂问题转化为已经解决的标准形式，是算法设计的重要思想。

------

## 四、去除前导零

减法结果可能高位为 0，比如 `100 - 99 = 001`，实际上结果应为 `1`。
代码用 `while` 循环去掉高位零：

cpp

```
while (ans[lans - 1] == 0 && lans > 1) --lans;
```



注意要保留至少一位（当结果为 0 时，输出一个 `0`）。

> **收获**：模拟竖式后，必须处理前导零，这是高精度题目的常见坑点。

## 五、与加法的对比，加深“模拟”思维

| 操作          | 加法                 | 减法                                 |
| :------------ | :------------------- | :----------------------------------- |
| 核心运算      | 对应位相加 + 进位    | 对应位相减 + 借位                    |
| 进位/借位处理 | 当前位 / 10 给下一位 | 当前位 < 0 时，下一位 -1，当前位 +10 |
| 额外步骤      | 处理最高位进位       | 处理符号、去除前导零                 |

通过对比，你能更深刻地体会到“模拟”的本质：**把人类计算的每一步，用变量和循环精确复现**。

------

## 六、边界情况的全面考虑

- 两个数相等（结果为 0，前导零去完后剩一个 0）
- 连续借位（如 `1000 - 1`）
- 被减数小于减数（需输出负号）
- 结果高位有多个零（循环去除）

这些边界你都要手动测试，确保代码健壮。

3. 模拟竖式（具体运算不同）
